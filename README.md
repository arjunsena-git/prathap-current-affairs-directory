# Prathap KM's Current Affairs Directory

A free static current-affairs website for Prathap KM's KAS and UPSC Civil Services preparation.

The site is generated by a Python scraper, committed as `index.html`, and served from GitHub Pages. There is no backend server, no login, no external API key, and no paid service.

## What It Does

- Scrapes free, publicly accessible current-affairs sources.
- Generates a colorful mobile-first HTML learning directory.
- Groups items into National, Karnataka/State, Economy, Science & Tech, Environment, and International.
- Keeps the last 7 days of content visible.
- Deduplicates headlines and sorts newest first.
- Shows source, date, readable in-page summary, source link, and source availability labels.
- Updates every 2 hours from 6:00 AM to 10:00 PM IST through GitHub Actions.
- Provides a public **Reload latest published** button that reloads the page without requiring GitHub access.

## Current Sources

- PIB RSS: `https://pib.gov.in/rssfeed`
- DD News RSS: `https://ddnews.gov.in/rss`
- The Hindu National RSS: `https://www.thehindu.com/news/national/feeder/default.rss`
- GKToday current affairs page: `https://www.gktoday.in/current-affairs/`
- KPSC notices: `https://kpsc.kar.nic.in/`
- Karnataka DIPR press updates: `https://karnatakavarthe.org/en/`

HTML sources are checked against `robots.txt` before scraping. If a source fails, the script logs the error, keeps going, and reuses cached content for that source with a `source unavailable` label.

## Fork And Host On GitHub Pages

1. Fork this repository.
2. In the fork, go to **Settings → Pages**.
3. Under **Build and deployment**, choose **Deploy from a branch**.
4. Select the branch you use for the site, usually `main`.
5. Select `/ (root)` as the folder.
6. Save. GitHub will publish the stable URL shown on that page.
7. Go to **Actions** and enable workflows if GitHub asks.

## Update Schedule

The workflow is in `.github/workflows/update-current-affairs.yml`.

```yaml
schedule:
  - cron: "30 0,2,4,6,8,10,12,14,16 * * *"
```

GitHub cron uses UTC. This schedule runs every 2 hours from 00:30 UTC through 16:30 UTC, which is 6:00 AM through 10:00 PM IST.

To change the time:

1. Convert the desired IST time to UTC.
2. Edit the cron line in `.github/workflows/update-current-affairs.yml`.
3. Commit and push the change.

Examples:

- 6:00 AM IST daily = `30 0 * * *`
- Every 2 hours from 6:00 AM to 10:00 PM IST = `30 0,2,4,6,8,10,12,14,16 * * *`
- Hourly all day = `30 * * * *`

## Manual Refresh

The public page's **Reload latest published** button reloads the latest GitHub Pages version. It does not require GitHub access and does not expose any token.

To force a new scrape as the repository owner, open the GitHub Actions workflow page and click **Run workflow**.

A true one-click browser trigger would require a GitHub token in client-side JavaScript, which would expose write access publicly. This project intentionally avoids that.

## Add More Sources

Edit `SOURCES` in `scripts/scrape_current_affairs.py`.

For RSS:

```python
Source("Source Name", "https://example.com/feed.xml", "rss", "National")
```

For HTML pages:

```python
Source("Source Name", "https://example.com/news/", "html", "Karnataka/State")
```

Then add parsing rules in `html_candidates()` if the site needs custom selectors. Prefer official RSS feeds whenever available.

## Notes

- Keep scraping polite: use public pages, avoid paid/paywalled content, and respect `robots.txt`.
- The generated site is static, bookmark-friendly, and works from one stable GitHub Pages URL.
- `data/current_affairs_cache.json` is committed so failed sources can show their last known content.
